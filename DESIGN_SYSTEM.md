# 기계공학과 학생회 웹사이트: 디자인 시스템 및 UI/UX 가이드라인

## 1. 개요

본 문서는 기계공학과 학생회 웹사이트의 일관된 디자인 및 사용자 경험(UI/UX)을 유지하기 위한 가이드라인을 제공합니다. Claude를 포함한 모든 개발자 및 디자이너는 본 가이드라인을 준수하여 웹사이트의 **'야무지고 전문적인'** 아이덴티티를 유지해야 합니다. 이 가이드라인은 웹사이트의 시각적 통일성을 확보하고, 사용자에게 최적의 경험을 제공하는 것을 목표로 합니다.

## 2. 디자인 철학

기계공학과 학생회 웹사이트의 디자인은 다음 세 가지 핵심 철학을 기반으로 합니다.

*   **전문성 (Professionalism)**: 신뢰감 있고 체계적인 정보 전달을 위해 깔끔하고 정돈된 레이아웃, 명확한 타이포그래피, 그리고 절제된 애니메이션을 지향합니다.
*   **기술적 미학 (Technical Aesthetics)**: 기계공학과의 특성을 반영하여 정교하고 현대적인 느낌을 부여합니다. 이는 컬러 팔레트, 배경 패턴, 아이콘 선택 등에서 드러납니다.
*   **사용자 중심 (User-Centric)**: 정보 접근성을 최우선으로 고려하며, 직관적인 네비게이션과 명확한 인터랙션 피드백을 통해 사용자가 쉽고 편리하게 정보를 얻을 수 있도록 합니다.

## 3. 컬러 팔레트

웹사이트의 주 색상은 **Rich Navy**와 **Steel Blue**를 중심으로 구성되며, 보조 색상으로 그린, 레드, 오렌지 등을 활용합니다. 모든 색상은 `globals.css`에 정의된 Tailwind CSS 커스텀 속성(`var(--primary)`, `var(--accent)` 등)을 통해 사용해야 합니다.

| :------------ | :--------------------- | :----------------------- | :----------------------- | :------------------------------------- |
| **주요 색상** | `--primary`            | `oklch(0.32 0.12 260)`   | `oklch(0.78 0.12 200)`   | 브랜드의 핵심 색상 (Rich Navy / Bright Steel Blue) |

## 4. 타이포그래피

폰트는 `next/font/google`의 `Geist` 폰트를 기본으로 사용하며, 가독성과 전문성을 최우선으로 고려합니다. 텍스트의 크기, 굵기, 줄 간격, 자간은 Tailwind CSS의 기본 유틸리티 클래스를 활용하되, `globals.css`에 정의된 `font-feature-settings` 및 `-webkit-font-smoothing` 속성을 통해 최적화된 렌더링을 보장합니다.

*   **제목 (Headings)**: `font-black`, `tracking-tight` 클래스를 사용하여 강렬하고 현대적인 느낌을 부여합니다.
*   **본문 (Body Text)**: `text-muted-foreground`, `leading-relaxed` 클래스를 사용하여 편안한 가독성을 제공합니다.
*   **코드 (Code)**: `bg-muted`, `px-2`, `py-1`, `rounded`, `text-xs`, `font-bold` 클래스를 사용하여 인라인 코드 블록을 명확하게 표시합니다.

## 5. 컴포넌트 가이드라인

### 5.1. Navbar (상단 네비게이션 바)

*   **스타일**: `glass-premium` 유틸리티 클래스를 사용하여 배경 블러 및 그림자 효과를 적용합니다. `border-b`를 통해 하단 경계선을 명확히 합니다.
*   **로고**: `Settings` 아이콘과 학과명, 학생회명을 조합하여 브랜드 아이덴티티를 강화합니다. 로고 클릭 시 `group-hover:rotate-12`와 같은 마이크로 인터랙션을 추가하여 생동감을 부여합니다.
*   **네비게이션 링크**: 활성 링크는 `text-primary bg-primary/10 shadow-md` 스타일을 적용하고, 비활성 링크는 `text-muted-foreground hover:text-foreground hover:bg-muted/60`를 사용하여 명확한 상태 변화를 보여줍니다. 각 링크에 아이콘을 포함하여 시각적 인지도를 높입니다.
*   **모바일 메뉴**: `lg:hidden`을 사용하여 데스크톱과 모바일 메뉴를 분리하고, 모바일 메뉴는 `animate-in slide-in-from-top-2` 애니메이션을 통해 부드럽게 나타나도록 합니다.

### 5.2. Footer (하단 푸터)

*   **스타일**: `border-t border-border/40 bg-muted/20`를 사용하여 상단 경계선과 은은한 배경색을 적용합니다.
*   **정보 구조**: 브랜드 정보, 주요 서비스 링크, 리소스 링크, 문의처 정보를 명확하게 구분하여 그리드 레이아웃으로 배치합니다.
*   **아이콘**: `Lucide-react` 아이콘을 사용하여 시각적 요소를 강화하고, `text-primary/60`와 같은 색상으로 브랜드 통일성을 유지합니다.
*   **저작권 및 정책**: 하단에 저작권 정보, 개인정보처리방침, 이용약관 링크를 포함하여 법적 고지 사항을 명확히 합니다.

### 5.3. Card (카드 컴포넌트)

*   **기본 스타일**: `rounded-2xl`, `border-border/60`, `shadow-lg shadow-primary/5`를 기본으로 사용하여 부드러운 모서리와 은은한 그림자 효과를 부여합니다.
*   **호버 효과**: `hover-lift-premium` 유틸리티 클래스를 사용하여 마우스 오버 시 카드가 살짝 떠오르는 듯한 애니메이션 효과를 적용합니다.
*   **콘텐츠 구성**: `CardHeader`, `CardTitle`, `CardDescription`, `CardContent`를 사용하여 콘텐츠의 계층 구조를 명확히 합니다.

### 5.4. Button (버튼 컴포넌트)

*   **기본 스타일**: `h-13`, `px-8`, `text-base`, `rounded-xl`, `font-semibold`를 사용하여 충분한 크기와 명확한 시각적 요소를 제공합니다.
*   **그림자**: `shadow-lg shadow-primary/30`를 사용하여 버튼에 깊이감을 더하고, 호버 시 `hover:shadow-primary/50`로 그림자 효과를 강화합니다.
*   **애니메이션**: `hover:scale-105 active:scale-95`를 통해 버튼 클릭 및 호버 시 미묘한 스케일 변화를 주어 인터랙션 피드백을 강화합니다.

### 5.5. Badge (뱃지 컴포넌트)

*   **스타일**: `badge-premium`, `badge-success`, `badge-warning`과 같은 커스텀 유틸리티 클래스를 사용하여 목적에 맞는 색상과 스타일을 적용합니다.
*   **아이콘**: 뱃지 내부에 작은 아이콘을 포함하여 정보의 종류를 시각적으로 빠르게 인지할 수 있도록 합니다.

## 6. 유틸리티 클래스 (Custom Utilities)

`globals.css`에 정의된 커스텀 유틸리티 클래스는 웹사이트의 디자인 아이덴티티를 유지하는 데 중요한 역할을 합니다. 다음 유틸리티 클래스를 적극적으로 활용해야 합니다.

*   `glass-premium`: 고급스러운 글래스모피즘 효과 (배경 블러, 그림자, 테두리)
*   `tech-mesh`: 기계공학적 느낌의 배경 메시 패턴
*   `premium-shadow`, `premium-shadow-lg`: 깊이감 있는 그림자 효과
*   `hover-lift-premium`: 마우스 오버 시 부드럽게 떠오르는 애니메이션
*   `smooth-transition`: 모든 CSS 속성에 대한 부드러운 전환 효과
*   `text-balance`: 텍스트 균형을 자동으로 맞춰주는 CSS 속성
*   `line-accent`: 제목 하단에 강조 라인을 추가하는 효과
*   `skeleton`: 로딩 상태를 표시하는 스켈레톤 애니메이션
*   `border-accent-top`, `border-accent-left`: 강조 색상의 테두리

## 7. 애니메이션 및 인터랙션

웹사이트 전반에 걸쳐 부드럽고 미묘한 애니메이션을 적용하여 사용자 경험을 향상시킵니다.

*   **페이드 인 (Fade-in)**: 페이지 로드 시 콘텐츠가 부드럽게 나타나도록 `animate-in fade-in` 클래스를 활용합니다.
*   **슬라이드 인 (Slide-in)**: 특정 요소가 아래 또는 옆에서 슬라이드하며 나타나도록 `slide-in-from-bottom` 또는 `slide-in-from-left` 클래스를 사용합니다.
*   **호버 효과 (Hover Effects)**: 카드, 버튼, 링크 등 인터랙션 가능한 요소에 `transition-all`, `duration-300`, `hover:scale-105`, `hover:shadow-lg`와 같은 클래스를 적용하여 시각적 피드백을 제공합니다.

## 8. 이미지 및 아이콘

*   **아이콘**: `lucide-react` 라이브러리의 아이콘을 사용합니다. 이모지 사용은 지양하며, 필요한 경우 `Settings`, `Bell`, `BookOpen` 등 의미에 맞는 아이콘을 선택합니다.
*   **이미지**: 웹사이트에 사용되는 모든 이미지는 고해상도여야 하며, 브랜드 컬러 팔레트와 조화를 이루어야 합니다. 불필요한 이미지 사용은 자제하고, 정보 전달에 필수적인 경우에만 사용합니다.

## 9. 접근성 (Accessibility)

모든 UI 요소는 웹 접근성 표준을 준수해야 합니다. 충분한 대비를 가진 색상, 키보드 네비게이션 지원, 의미론적 HTML 마크업 등을 통해 모든 사용자가 웹사이트를 편리하게 이용할 수 있도록 합니다.

## 10. 개발 가이드라인

*   **Tailwind CSS**: 모든 스타일링은 Tailwind CSS 유틸리티 클래스를 사용하여 인라인으로 적용하는 것을 원칙으로 합니다. 필요한 경우 `globals.css`에 커스텀 유틸리티를 추가할 수 있습니다.
*   **Shadcn/UI**: `shadcn/ui` 컴포넌트를 기반으로 개발하며, 필요에 따라 커스터마이징하여 사용합니다.
*   **반응형 디자인**: 모든 페이지는 데스크톱, 태블릿, 모바일 등 다양한 화면 크기에서 최적화된 레이아웃을 제공해야 합니다.
*   **코드 재사용**: `components` 폴더 내의 재사용 가능한 컴포넌트를 적극적으로 활용하여 개발 효율성을 높이고 일관성을 유지합니다.
